# design.md

**概要**
- 目的: `req.md` の要件（アイドルグッズEC、コンビニ店頭支払／自社発行コード、店舗受取）を踏まえ、参照サイトのモバイル重視のビジュアル表現を取り入れた画面デザイン仕様を定義する。参照サイトはモバイルファーストでビジュアルを強調するランディング／物販ページであり、本仕様ではその印象（大胆なヒーロー、視覚的インパクト、明確なCTA）を再解釈して適用する。

**デザイン原則**
- モバイルファースト: スマートフォンでの閲覧・購入体験を最優先に設計。操作は親指で届く範囲を基準にする。
- 視覚的インパクト: 大きなヒーロー画像、コントラストの強いCTA、商品画像の強調で感情を喚起する。
- シンプルで迅速な購入: カート〜チェックアウト〜コード表示に至るステップを最小化し、迷わせない導線を作る。
- ブランド一貫性: ダークトーンやコントラストの高いアクセント色でライブ/イベントの熱量を表現する。
- 安心感の提示: 支払い期限や受取時の身分証要件を明確に提示し、ユーザ不安を軽減する。

**カラーパレット（例）**
- 基調: 深黒 #0b0b0b (背景)
- サブ背景: ダークグレー #121212
- テキスト: 白 #FFFFFF / サブテキスト #CFCFCF
- アクセント (CTA): 血色系レッド #E13B3B または コーラル #FF5A5F
- 成功/確認: 緑 #2ECC71
- 注意: 黄色 #F1C40F

**タイポグラフィ**
- 見出し: 太めのサンセリフ（例: Inter Bold / Noto Sans JP Bold）
- 本文: 読みやすいサンセリフ（例: Noto Sans JP Regular）
- サイズ感（モバイル）: H1 28–32px、H2 20–24px、本文 14–16px、ボタン 16–18px

**レスポンシブのブレークポイント（推奨）**
- モバイル: 0–599px（主要対象）
- タブレット: 600–1023px
- デスクトップ: 1024px+

**画面ごとのレイアウト要件（要素一覧／優先度）**

- Home（ホーム）
  - ヒーローバナー（イベント告知／ビジュアル重視）: フル幅、中央に短いキャッチと1つの主要CTA（例: 今すぐグッズを見る）。
  - 注目イベントカード: スクロール横スライダー（タッチでスワイプ）。
  - 商品カルーセル/おすすめ: 大きめのサムネイル（1列〜2列）、商品名・価格・即買ボタン。

- Category / Listing（商品一覧）
  - 検索窓（上部固定）
  - 絞り込みトグル（イベント / 価格帯 / ランダム封入など）
  - 商品カード: 画像（1:1または4:5）、商品名、価格、在庫バッジ（在庫少→黄色）、簡易「カートに追加」ボタン。

- Product Detail（商品詳細）
  - 画像ギャラリー（スワイプ対応、拡大可）
  - 価格・在庫・説明・属性（サイズ・仕様）
  - CTA群: 数量セレクタ、カートに追加（大きなCTA）、今すぐ購入（即座にチェックアウト）
  - レビュー/注意事項（受取時身分証の注意文はチェックアウト側でも再提示）

- Search（検索結果）
  - 結果リスト + フィルタパネル（モーダルで表示）

- Cart（カート）
  - 商品一覧（小さなサムネイル、数量変更）、合計、受取方法選択（配送 or コンビニ受取）。
  - 「購入手続きへ」ボタンは常時画面下固定で、色はアクセント。

- Checkout（チェックアウト）
  - 入力フォームはステップ分割（1: 連絡先 / 2: 受取方法・店舗選択 / 3: 確認）を想定
  - 受取方法で「コンビニ受取」を選択すると、チェーン選択（セブン／ローソン／ファミマ）と店舗コード入力／検索UIを表示
  - 注文確定ボタンは強調。支払い有効期間（3日）を明示して注意喚起。

- Payment Code Display（支払いコード表示）
  - 画面上部に大きくコード（フォントサイズ可変）
  - QRコード（店舗が読み取る場合）とコード文字列を併記
  - 支払い期限（残り時間のカウントダウン）表示
  - 「印刷」ボタン（PC向け）と「店頭で提示」指示（モバイルは画面提示推奨）
  - 支払い手順（短い箇条書き）と受取時の身分証要件を明記（運転免許証／マイナンバー／パスポート）

- Order Confirmation（注文完了）
  - 注文番号、受取店舗、支払い期限を明示。
  - メール送付の案内と「支払いコードを見る」ボタンを用意。

- Account / My Page（マイページ）
  - 注文履歴一覧（支払状況・受取状況）、身分証の事前登録（任意）リンク、通知設定。

- Admin Dashboard（管理者）
  - 受注テーブル（フィルタ: 未払い/支払済/受取済/キャンセル）
  - 支払いコードの発行ログ／在庫アラート／受取確認ボタン

**主要コンポーネント仕様**
- ヘッダー
  - 左上: ハンバーガーメニュー（ナビ）、中央: ブランドロゴ、右上: カートアイコン（バッジで数量表示）
- ヒーロー
  - フルスクリーン幅、高コントラストテキストを重ねる（暗めのオーバーレイで可読性確保）
- 商品カード
  - 画像領域（16:19 or 4:5推奨）、下部に商品名、価格、右下に小さな「カートに追加」ボタン
- ボタン
  - プライマリ（CTA）: アクセント色、角丸4–8px、サイズ大（指で押しやすい）
  - セカンダリ: 低コントラストで補助的操作
- 在庫バッジ
  - 在庫あり: グレー/薄緑、 残少: 黄色、 在庫なし: 赤（購入不可）

**インタラクション・アニメーション**
- タッチスワイプでのカルーセル（慣性スクロール）
- CTAタップ時の即応フィードバック（リップル/スケール）
- 支払い期限のカウントダウンはリアルタイムで更新（残時間が短くなると色で強調）
- モーダルやフライアウトはスクロールをロックして重点化

**アセット要件**
- ヒーロー画像: 16:9 以上、最小幅 1200px 推奨（サーバーでは複数解像度を用意）
- 商品画像: 800x800px（正方形）または 800x1000px（縦長）を推奨、背景は透過でなく単色推奨
- アイコン: SVG を使用、ファビコンとソーシャル画像も用意

**アクセシビリティ**
- コントラスト比は WCAG AA を満たす（CTA と背景）
- 画像に代替テキストを必須化
- フォームはラベルとaria属性を付与
- キーボード操作でのフォーカス順を確認

**実装ノート（技術的推奨）**
- フロント実装: Next.js / React + Tailwind CSS の組み合わせを推奨（モバイル最適化とコンポーネント再利用が容易）。代替として Vue + Nuxt も可。
- CSS: BEM や utility-first（Tailwind）で管理。ダークテーマを基本にアクセントカラーを変数で管理。
- 画像配信: CDN（画像はレスポンシブに切替）
- 支払いコード画面: サーバでコード画像（QR含む）を生成してキャッシュ。表示はプリント対応。

**運用・UX上の注意点**
- コード発行直後に在庫を確保する場合、在庫戻しの自動化（期限切れでキャンセル）が不可欠。
- 支払い期限のリマインド通知（メール/SMS）を実装し、期限切れ率を下げる。
- 受取時のID確認フローは店舗スタッフの運用負担になるため、事前通知・店舗向け簡易マニュアルを用意する。

**参考（デザイン上の特徴抽出）**
- 参照サイトはモバイル表示でビジュアルを強く見せる設計（大きな画像、シンプルな導線、明瞭なCTA）。当仕様ではその印象を踏襲しつつ、購入の安全性（支払い手順やID確認）をUX上強調する。
